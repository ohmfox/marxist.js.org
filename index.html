The Classless Javascript Manifesto

Down with Javascript classes!

ES6 added the class keyword to Javascript. We think that this adds far too much complexity to Javascript and that the built in "class-like" abilities of Javascript are much more clear and powerful. We are proposing that Javascript developers avoid the class keyword in favor of more powerful alternatives.

An example for the masses!

Take a look at this code. 

        
          var aClass = arg => class {
            constructor() {
              console.log(arg);
            }
          };
          new (aClass("ey"))(); //prints "ey" and returns an object

At first glance this looks pretty weird. There is a closure, an arrow function, a class keyword, However, with a little work (converting the class to function) this code becomes easy to read and understand. 

        
          var aClass = (arg) => {
            return function() {
              console.log(arg);
              return {};
            }
          }
          new (aClass("ey"))(); // prints "ey" and returns an object

So what are you saying?

Basically, there are lots of good reasons not to use the Javascript class keyword but the most important is that the class keyword hides a lot of the behind the scenes parts of Javascript and abstracts away from what is actually happening. It is much better to use a function or an object and work directly with the prototype.
